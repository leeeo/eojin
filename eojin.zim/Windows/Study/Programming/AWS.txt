====== AWS ======

===== Amazon Web Services =====


AWS와 함께 확장성 높은 천만 사용자 웹 서비스 만들기
클라우드를 통한 확장성 높은 웹 및 모바일 앱 개발을 위한 아키텍쳐 단계별 구성 방법!


===== 클라우드 컴퓨팅의 이점 =====
선투자금 없음 - 고정비용에서 가변비용으로
규모의 경제 - 단일회사 이상의 TCO절감을 가격에 반영
탄력적인 처리용량 - 필요한 처리용량 예측 불필요 오버프로비저닝 방지
속도와 민첩성(agility) - 인프라를 주단위가 아닌 분 단위로 프로비저닝
비지니스에 집중 - 핵심 서비스에만 집중 가능
글로벌 진출 - 수분 내에 글로벌 고객들을 위한 서비스 런칭

싱글 EC2 instance
- 한 서버에 풀스택 가능
	웹 서버
	데이터베이스
	시스템 운영
싱글 Elastic IP
DNS : Amazon Route 53

Amazon Route 53
- 가용성과 확장성이 뛰어난 클라우드 DNS
- 동적으로 사용자에게 노출될 DNS 레코드 타입과 값 조정
- 각종 다양한 로드밸러싱 기능 지원

관계형 데이터베이스
- 기존에 개발된 애플리케이션을 사용하는 경우
- 다양한 샘플, 커뮤니티 및 기술 정보 습득 가능
- RDB를 통해서도 천만 사용자 확장 가능

NoSQL를 선택하는 경우?
- 메타데이터나 로그 기반 데이터 셋
- 데이터 속성상 관계(Relation)이 없는 경우
- 응답률이 빠른 대용량 데이터를 다루는 경우(테라급)
- 데이터를 대량으로 추가하는 경우(초당 수천건)

Amazon [[DynamoDB]]
- 빠르고 성능 예측이 가능한 분산 [[NoSQL]]
	평균 서비스 지연 시간은 보통 수 밀리초 미만
	문서 데이터 구조와 키 값 데이터 구조를 모두 지원
- 확장 및 용량 등 매니지드 서비스로 제공
	HDD가 아니라 SSD 기반이라 빠른 속도
	경험이 적은 운영 노하우 부분의 걱정을 덜 수 있음
